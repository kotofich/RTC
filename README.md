# Внутренняя отчетность по ОС серверов

Веб-приложение на **Python (Django)** + **PostgreSQL** для внутреннего контура.

## Что реализовано

- Отчеты построены по главным колонкам листа **"ОС серверов"** из файла
  `Комплексный опросник (SIEM сканер уязвимостей)_v2.6_Голос 16.04.2026.xlsx`.
- У каждого пользователя есть **личный кабинет**: таблица его отчетов, где
  каждая строка — отдельный отчет, каждое поле — отдельный столбец.
- Слева у каждой строки есть кнопка **"изменить"**.
- В поле `Роль/Функция` выбор идет из списка сущностей, основанных на листах Excel.
- В ЛК добавлены листы (вкладки) по сущностям: на каждом листе только IP-адреса
  отчетов соответствующей сущности.
- Права доступа:
  - обычный пользователь: создает и редактирует только свои отчеты;
  - администратор (`is_staff`/`superuser`): только чтение.
- Регистрация пользователя через модальное окно на странице входа.
- Отделы и отделенческая логика убраны из пользовательского сценария.

## Быстрый запуск

### 1) Поднять PostgreSQL

```bash
docker compose up -d db
```

### 2) Установить зависимости

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3) Настроить окружение

```bash
cp .env.example .env
```

### 4) Применить миграции

```bash
python manage.py migrate
```

### 5) (Опционально) заполнить демо-данными

```bash
python manage.py seed_demo_data
```

### 6) Запустить сервис

```bash
python manage.py runserver 127.0.0.1:8002
```

## Демо-аккаунты

- `admin / Admin123!` — администратор (только чтение)
- `employee / Employee123!` — пользователь (создание/изменение)

## Ключевые URL

- `/accounts/login/` — вход
- `/accounts/register/` — регистрация
- `/lk/` — личный кабинет (основная ручка)
- `/cabinet/` — legacy-алиас на ЛК
- `/reports/new/` — создание отчета
- `/reports/<id>/edit/` — редактирование отчета
- `/admin/` — Django admin
